Сравнение временных затрат для стандартного алгоритма merge_sort:

Random Arrays:
С увеличением размера массива время выполнения растет логарифмически, что соответствует ожидаемой временной сложности. Стандартный merge_sort показывает стабильное поведение на случайных массивах.

Reversed Arrays:
Merge_sort также справляется с обратно отсортированными массивами, но его время выполнения заметно выше по сравнению с случайными массивами. 

Almost Sorted Arrays:
Время выполнения на почти отсортированных массивах остается относительно низким, что подтверждает эффективность алгоритма в случае частично упорядоченных данных.

Сравнение временных затрат для гибридного алгоритма merge_insertion_sort:

Random Arrays:
При использовании гибридного алгоритма для небольших массивов можно заметить улучшение времени выполнения по сравнению со стандартным merge_sort. С увеличением размера входного массива в insertion_sort время выполнения сначала уменьшается, но затем начинает увеличиваться. 

Reversed Arrays:
Гибридный алгоритм также показывает улучшение на обратно отсортированных массивах при небольших размерах.

Almost Sorted Arrays:
Гибридный алгоритм хорошо справляется с почти отсортированными массивами, особенно при небольших размерах.

Общие выводы:
Гибридный алгоритм действительно может быть эффективным на небольших массивах или массивах, близких к упорядоченным. Стандартный merge_sort остается надёжным алгоритмом для сортировки в широком диапазоне размеров входных данных.
